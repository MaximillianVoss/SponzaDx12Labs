# Лабораторная работа 1

## Что требовалось

Собрать базовое `Win32 + DirectX 12` приложение: окно, цикл сообщений, инициализация `Device`, `SwapChain`, командной очереди, корневой сигнатуры и первого графического `PSO`. На этом этапе закладывалась основа для всех следующих работ.

## Что было сделано

- создано одно общее приложение `SponzaDx12Labs`, внутри которого все лабораторные наращиваются последовательно;
- настроено окно `Win32`, собственная `WndProc`, цикл обработки сообщений и базовая логика `D3DApp`;
- инициализированы основные объекты `DirectX 12`: устройство, очередь команд, swap chain, RTV/DSV/SRV-дескрипторы, command list;
- добавлены ресурсы кадра и константные буферы для объектов, материалов и прохода;
- собран базовый forward-rendering проход;
- подключены шейдеры общего назначения и первая корневая сигнатура;
- подготовлена загрузка сцены из `OBJ/MTL` и текстур из `TGA`;
- в качестве основной сцены подключена `Sponza`.

## Что было добавлено в проект

- решение и проект `Visual Studio`: `SponzaDx12Labs.sln`, `SponzaDx12Labs.vcxproj`;
- основной класс приложения: `SponzaLabsApp.h`, `SponzaLabsApp.cpp`;
- разбиение логики по файлам:
  `SponzaLabsAppBuild.cpp` для инициализации,
  `SponzaLabsAppRender.cpp` для рендера,
  `SponzaLabsAppUpdate.cpp` для обновления состояния;
- ресурсы кадра: `FrameResource.h`, `FrameResource.cpp`;
- загрузчики ресурсов: `ObjModelLoader.h`, `ObjModelLoader.cpp`, `TgaTextureLoader.h`, `TgaTextureLoader.cpp`;
- базовые шейдеры: `Shaders/Common.hlsl`, `Shaders/Default.hlsl`.

## Какие файлы отвечают за эту работу

- `SponzaLabsApp.cpp`:
  старт приложения, инициализация, стартовая камера, окно, caption;
- `SponzaLabsAppBuild.cpp`:
  загрузка сцены, текстур, корневой сигнатуры, descriptor heap, материалов и геометрии;
- `SponzaLabsAppRender.cpp`:
  базовый forward pass и вызовы `DrawIndexedInstanced`;
- `FrameResource.*`:
  буферы кадров и данные для GPU;
- `ObjModelLoader.*`:
  загрузка `OBJ/MTL`, разбор мешей и материалов;
- `TgaTextureLoader.*`:
  загрузка текстур для материалов.

## Что можно показать преподавателю

- приложение запускается как обычное `Windows Desktop Application`;
- есть живая 3D-сцена, а не статическая картинка;
- работает управление камерой `W/A/S/D` и мышью;
- модель и материалы загружаются не “зашитыми”, а из внешних файлов `Assets/Sponza-master`.

## Что можно сказать при защите

- выбран один общий проект, потому что по ТЗ лабораторные логически связаны и каждая следующая работа расширяет предыдущую;
- база сделана на низком уровне `Win32 + DX12`, без `WPF/WinForms/UWP`, потому что именно это и требуется в задании;
- сцена `Sponza` используется как реальный тестовый набор мешей и материалов, на котором удобно проверять освещение, deferred rendering, тесселяцию и culling;
- на первом этапе была заложена архитектура, чтобы дальше не переписывать проект заново под каждую следующую лабораторную.
